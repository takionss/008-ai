## <span style="color: #27AE60;">인공지능의 한계를 넘어서는 정교한 명령 설계와 구조화의 기술</span>




인공지능에게 "이 사이트 정보를 긁어줘"라고 말하는 것과, 사이트의 골격을 미리 파악해 세부적인 지침을 주는 것은 하늘과 땅 차이입니다. 제가 처음 인공지능을 도구로 활용할 때 범했던 가장 큰 실수가 바로 너무 막연하게 질문을 던졌던 것이었어요. 그러면 인공지능은 아주 뻔하고 교과서적인 코드만 내놓기 일쑤입니다. 실전에서 살아남는 코드를 얻으려면, 여러분은 인공지능에게 웹사이트의 '지도'를 먼저 그려주어야 합니다.

예를 들어, 단순히 URL 하나만 던져주지 마세요. 여러분이 수집하고자 하는 정보가 담긴 짧은 HTML 코드 조각을 복사해서 인공지능에게 보여주며 "이 태그 구조 안에서 텍스트 정보만 추출해줘"라고 명령해 보세요. 이렇게 구체적인 단서를 제공하면 인공지능은 훨씬 정교한 선택자(Selector)를 찾아냅니다. 또한, 데이터 수집 과정에서 발생할 수 있는 변수들을 미리 알려주는 것도 중요합니다. "페이지가 넘어갈 때 데이터 로딩 시간이 걸릴 수 있으니, 특정 요소가 나타날 때까지 기다리는 로직을 넣어줘"라고 콕 집어 말하는 식이죠.

여기서 제가 실무에서 가장 효과를 보았던 프롬프트 구성 팁 3가지를 정리해 드릴게요. 이 방식만 익혀도 여러분이 받는 코드의 질이 달라질 것입니다.

## <span style="color: #D35400;">변화무쌍한 웹 환경에서 살아남는 유연하고 지속 가능한 코드 관리법</span>




인공지능이 짜준 코드가 오늘 작동한다고 해서 내일도 작동하리라는 보장은 없습니다. 웹사이트는 생물과 같아서 자고 일어나면 구조가 바뀌어 있기 마련이죠. 많은 분이 인공지능 덕분에 코딩이 쉬워졌다고 좋아하지만, 사실 진짜 실력은 코드가 멈췄을 때 이를 얼마나 빠르게 복구하느냐에서 드러납니다. 제가 현업에서 수집 자동화 시스템을 구축할 때 가장 공을 들이는 부분도 바로 이 '지속 가능성'입니다.
